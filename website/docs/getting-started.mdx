## Your first form

Below is a minimal but complete example using **submit-based validation**.

```tsx
import { useForm } from "formora";

function LoginForm() {
  const form = useForm({
    initialValues: {
      email: "",
      password: "",
    },
    validateOn: "submit",
  });

  return (
    <form onSubmit={form.handleSubmit((values) => console.log(values))}>
      <div>
        <label>Email</label>
        <input
          {...form.register("email", {
            validate: (value) => (!value ? "Email is required" : undefined),
          })}
        />
        {form.shouldShowError("email") && <p>{form.errors.email}</p>}
      </div>

      <div>
        <label>Password</label>
        <input
          type="password"
          {...form.register("password", {
            validate: (value) =>
              value.length < 6 ? "Minimum 6 characters" : undefined,
          })}
        />
        {form.shouldShowError("password") && <p>{form.errors.password}</p>}
      </div>

      <button type="submit">Sign in</button>
    </form>
  );
}
```

---

## How validation works here

:::info
Validation runs **only when the form is submitted** because `validateOn` is set to `submit`.
:::

- No errors are shown while typing
- Errors appear only after the first submit
- Error visibility is controlled by `shouldShowError`

This makes validation behavior **explicit and predictable**.

## Changing validation behavior

Formora supports three validation modes:

- `submit` â€“ validate on submit only
- `change` â€“ validate on every change
- `blur` â€“ validate when a field loses focus

You can switch modes by changing one option:

```ts
const form = useForm({
  validateOn: "blur",
});
```

Each mode is explained in detail in the **Validation Modes** section.
